# Diagnostic Maturity

Self-diagnosis has four stages.

## 1. Detect

Notice a stall, loop, false gate, retention problem, bad label, or queue paralysis.

## 2. Interpret

Identify the mechanism, not merely the symptom.

## 3. Correct

Create a correction path: doctrine, board change, schema change, packet, test, migration, or decision card.

## 4. Re-diagnose

Run the loop again when the same smell returns or when a new fuzzy category appears.

## Guardrail

Diagnosis is not completion.

A diagnosis is complete only when it produces one of:

- a tested correction;
- a dispatched correction packet;
- an explicit decision card;
- a parked item with owner, reason, and wake condition.
