# Quickstart

Ostler is a specification, not an application. There is nothing to install and
nothing to run. This "quickstart" is a **reading path**: the fastest way to
understand the model end to end.

Total time: about 10–15 minutes.

## Read in this order

1. **`docs/concepts.md`** (~4 min) — the narrative overview. Start here to build
   a mental picture before the formal spec.
2. **`docs/spec.md`** (~6 min) — the authoritative specification. The v0.1
   section is the part to read closely.
3. **`docs/glossary.md`** (~1 min) — keep open as a reference while reading.
4. **`examples/worked-example.md`** (~3 min) — watch one fuzzy "blocked" item get
   decomposed and moved. This is where the model clicks.
5. **`doctrine/`** (~1 min each) — five short, standalone rules. Read them
   individually; each stands on its own.

## The five ideas to take away

If you remember nothing else:

1. **Work is a durable estate, not a conversation.** It must survive any single
   session, operator, or agent.
2. **Gate type and actor are separate axes.** *What* is blocked is not *who* must
   act.
3. **Absorb gates before declaring "blocked."** Do all gate-free prep first;
   reduce the human ask to its smallest atom.
4. **The cockpit coordinates; workers do the work.** Keep the coordinator thin.
5. **Evidence and review over silent action.** Nontrivial work returns evidence
   and gets a second set of eyes.

## What you cannot do yet

- There is no reference implementation.
- There are no schemas, prompts, or scripts.
- `docs/reference-shape.md` sketches what an implementation *might* contain, but
  it is illustrative only — not a blueprint and not real code.

When the vocabulary, schema, and examples are stable, an implementation may
follow. For now, the deliverable is the model itself.
